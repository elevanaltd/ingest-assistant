# Sessions Directory

This directory contains session recordings, notes, and context from development sessions.

## Purpose
- Agent session recordings
- Development session notes
- Context preservation for multi-session work
- Debugging session logs
- Planning session outcomes

## Naming Convention
- `YYYY-MM-DD-{TOPIC}/` (session folders)
- `{SEQ}-{PHASE}-{ROLE}-{DESC}.md` (session documents)
- Example: `2025-11-11-video-transcoding/001-B2-implementation-lead-ffmpeg-integration.md`

## Structure
Session folders should contain:
- Session goals and outcomes
- Key decisions made
- Links to created artifacts
- Next steps and blockers
